Azure Monitoring Grafana (ARM)

ARM template for adding an VM with a fully automated bootstrap script to create monitoring that automatically connects to Azure monitor (data source) for metrics and allows users easy viewing of key metrics for Azure resources.

Resources used:

  • ARM template
    • Resource group
    • Virtual network
    • Network Interface
    • Network security group
    • Virtual machine
    • Public IP address
    • Disk
  • Bootstrap script for installing and running docker-compose
  • docker-compose.yml for Cloud Monitoring with Grafana, InfluxDB, and Telegraf
  • Configuration for Telegraf and InfluxDB
  • Custom dashboard for Grafana using CloudWatch and InfluxDB

Grafana provisioned Data sources:

  • Azure Monitor for getting metrics on Azure resources. Using Managed Identity.
  • InfluxDB data source in combination with Telegraf for getting metrics on service UpTime.

1. ARM template

ARM template for creating resources on Azure needed to start monitoring metrics.

2. docker-compose.yml for Grafana, InfluxDB, and Telegraf

Using docker-compose to be able to easily run Monitoring.

  • Grafana for visualizing data
  • InfluxDB for querying data
  • Telegraf for data aggregation in order to get service uptime metric.

3. Grafana datasource for Azure monitor and InfluxDB

Provision Grafana data source to be able to connect Grafana with Azure monitor Datasource and InfluxDB data source.

4. Configure Grafana, InfluxDB and Telegraf

  • “default.ini” (Grafana) uncomment default_home_dashboard_path to be able to start Grafana with a custom dashboard.
  • “datasource.yml” (Grafana) adds InfluxDB and Azure monitor configuration for data sources to be able to connect to them.
  • “telegraf.conf” (Telegraf) edit inputs.ping to be able to get service uptime.

5. Dashboard for Grafana

Provision Grafana dashboard to be able to have all the metrics.

6. Access control (IAM) - Subscription or Resource Group

Add role assignment: Monitoring Reader (Can read all monitoring data.)
